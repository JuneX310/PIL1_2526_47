Guide d'Architecture de MentorLink

Comprendre le fonctionnement global de Django, la communication Frontend-Backend et la gestion de MySQL.

1. Le fonctionnement de Django (Le patron MTV)

Django est basé sur le modèle de conception MTV (Model-Template-View). C'est une variante du modèle MVC (Model-View-Controller) classique :

Model (M)

Il définit la structure des données sous forme de classes Python dans models.py. C'est l'interface avec la base de données (MySQL). Django se charge de traduire le code Python en requêtes SQL.

Template (T)

C'est la partie visuelle (HTML). Dans MentorLink, vos gabarits sont situés dans frontend/templates/ (ex. base.html, explorer.html). Ils génèrent la structure HTML envoyée au navigateur.

View (V)

C'est le contrôleur. Écrites dans views.py, les vues contiennent le code Python qui récupère les requêtes du client, interagit avec la base de données via les modèles, et renvoie soit une page HTML, soit du JSON.

Dans votre projet : Lorsque vous visitez l'URL /matching/explorer/, le routeur (urls.py) appelle la fonction de vue explorer() dans views.py. Cette vue rend le template groupe1/explorer.html.

2. La communication Frontend & Backend

Pour le projet MentorLink, le frontend utilise Vue.js pour rendre la page dynamique, et communique avec le backend Django via des requêtes HTTP (AJAX).

Prenons l'exemple de la page Publier (publier.js) où un utilisateur soumet une annonce :

frontend/static/js/publier.js
JavaScript JS
const submitForm = () => {

    // 1. Récupération du jeton CSRF obligatoire pour sécuriser la requête POST

Côté backend, Django intercepte cette requête POST, décode le JSON reçu et renvoie une réponse JSON (JsonResponse) :

backend/apps/matching/views.py
Python PY
import json
from django.http import JsonResponse
from django.shortcuts import render
from .models import Annonce

def publier(request):
    if request.method == 'POST':
        try:
            # Décodage des données JSON envoyées par Vue.js
            data = json.loads(request.body)
            
            # Enregistrement dans la base MySQL grâce à l'ORM Django
            annonce = Annonce.objects.create(
                type=data.get('type'),
                sujets=data.get('subjects'),
                format=data.get('format'),
                creneaux=data.get('slots'),
                description=data.get('description')
            )
            
            return JsonResponse({'status': 'success', 'message': 'Annonce créée avec succès !'})
        except Exception as e:
            return JsonResponse({'status': 'error', 'message': str(e)}, status=400)
            
    # Si c'est une requête GET, on affiche simplement le formulaire
    return render(request, 'groupe1/publier.html')

3. La création de tables dans MySQL (Django ORM)

Dans un projet classique, on écrit du code SQL à la main pour créer des tables. Avec Django, on utilise un ORM (Object-Relational Mapper). On décrit les tables en Python, et Django génère et exécute automatiquement les requêtes SQL MySQL.

Voici la démarche étape par étape de comment les tables sont configurées et créées :

Étape A : Configuration de MySQL dans Django

La connexion est définie dans backend/config/settings.py sous la variable DATABASES. Elle récupère les variables d'environnement configurées dans le fichier .env :

Étape B : Déclaration du Modèle en Python

Chaque table est déclarée sous forme de classe Python dans le fichier models.py de l'application correspondante.

backend/apps/matching/models.py
Python PY
from django.db import models

class Annonce(models.Model):
    TYPE_CHOICES = [
        ('propose', 'Proposer du mentorat'),
        ('cherche', 'Chercher un mentor'),
    ]
    
    # Définition des colonnes de la table MySQL
    type = models.CharField(max_length=10, choices=TYPE_CHOICES)
    sujets = models.JSONField()            # Stocke les matières sélectionnées sous forme JSON
    format = models.CharField(max_length=20, default='présentiel')
    creneaux = models.JSONField()          # Liste des créneaux sélectionnés
    description = models.TextField()       # Texte descriptif long
    cree_le = models.DateTimeField(auto_now_add=True) # Horodatage automatique

    def __str__(self):
        return f"{self.type} - {self.sujets}"

Étape C : Génération et Application des Migrations

Une fois les classes écrites, on utilise les commandes de la console Django pour créer et appliquer les tables MySQL :

  • python manage.py makemigrations : Analyse le fichier models.py et génère un fichier de migration Python (dans le dossier migrations/). C'est un plan d'action qui liste les changements apportés aux modèles.
  • python manage.py migrate : Exécute concrètement les instructions SQL correspondantes sur la base MySQL configurée. C'est à cette étape que la table (par exemple matching_annonce) est créée dans MySQL.

Équivalence SQL générée automatiquement par Django :

CREATE TABLE matching_annonce (
    id INT AUTO_INCREMENT PRIMARY KEY,
    type VARCHAR(10) NOT NULL,
    sujets JSON NOT NULL,
    format VARCHAR(20) DEFAULT 'présentiel',
    creneaux JSON NOT NULL,
    description TEXT NOT NULL,
    cree_le DATETIME DEFAULT CURRENT_TIMESTAMP
);

4. Cycle de vie d'une requête dans MentorLink

Pour illustrer, voici comment se déroule l'interaction utilisateur de bout en bout lorsqu'un étudiant crée une annonce de mentorat :

  1. L'utilisateur remplit le formulaire sur l'interface frontend (HTML rendu par le template Django publier.html) et clique sur Publier.
  2. Le script JavaScript Vue.js (publier.js) intercepte le clic, récupère les données de formulaire et le jeton CSRF généré par Django.
  3. Vue.js effectue une requête fetch() POST asynchrone contenant le corps en JSON vers /matching/publier/.
  4. Le serveur Django reçoit la requête :
    • Il vérifie la validité du jeton CSRF.
    • La route /matching/publier/ redirige vers la fonction publier() de views.py.
    • La vue extrait les données JSON du corps de la requête.
  5. L'ORM Django instancie l'objet Annonce avec les données et exécute en arrière-plan la requête INSERT INTO sur le serveur MySQL.
  6. La base de données MySQL enregistre l'annonce et renvoie l'identifiant (ID) nouvellement créé.
  7. La vue Django renvoie une réponse JSON JsonResponse({'status': 'success'}).
  8. Vue.js reçoit la réponse du serveur, réinitialise le formulaire et affiche le Toast de confirmation à l'étudiant.

5. Comment lancer l'application

Voici la procédure pas à pas pour exécuter l'application MentorLink sur votre machine locale (sous Windows) :

Étape 1 : Ouvrir le terminal

Ouvrez PowerShell ou l'invite de commandes Windows, et placez-vous à la racine du projet :

cd "C:\Users\HP\Desktop\MY RESEARCH\PROJET-INTEGRATEUR-2025-2026-IFRI\CODE SOURCE\PIL1_2526_47"

Étape 2 : Activer le venv

Activez l'environnement virtuel Python pour charger les dépendances :

venv\Scripts\activate

Un badge (venv) s'affiche au début de la ligne.

Étape 3 : Aller dans backend

Naviguez vers le dossier contenant le fichier manage.py :

cd backend

Étape 4 : Lancer le serveur

Démarrez le serveur de développement de Django :

python manage.py runserver

🌐 Accéder au projet : Une fois le serveur démarré, ouvrez votre navigateur et allez sur http://127.0.0.1:8000.
