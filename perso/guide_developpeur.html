Objectif Principal de la plateforme

Faciliter la mise en relation entre mentors (étudiants plus anciens) et mentorés (nouveaux ou en difficulté) à l'IFRI. Grâce à un algorithme de matching intelligent, les profils sont automatiquement analysés par compétences, filières et disponibilités pour suggérer la meilleure collaboration académique possible.

Structure et organisation globale

Comment s'organise l'équipe de projet ?

Le projet est découpé en trois groupes fonctionnels qui développent leurs fonctionnalités de manière parallèle :

  • Groupe 1 — Matching & Exploration : Développe la logique de recommandation, le matching automatique et l'exploration des offres de mentorat.
  • Groupe 2 — Authentification : Gère la création de compte, la connexion sécurisée, la réinitialisation de mot de passe et les sessions utilisateurs.
  • Groupe 3 — Profils & Navigation : Développe les profils détaillés (compétences, filières, disponibilités), le tableau de bord (dashboard) et les flux de navigation généraux.
Où trouver la configuration et les guides locaux ?

Les fichiers de configuration locaux se situent dans backend/.env (à copier depuis backend/.env.example). Le fichier principal de documentation est le README.md à la racine du projet.

Méthodologie Moderne

Travailler Efficacement avec les IA

L'utilisation d'IA (Gemini, ChatGPT, Claude) peut accélérer le développement de MentorLink par 10, à condition de savoir structurer ses demandes et d'éviter les pièges fréquents liés à Django.

Règle d'Or Absolue

Django régit la base de données. Ne laissez jamais une IA générer un script SQL brut pour modifier les tables (ex: ALTER TABLE...) directement dans phpMyAdmin. Demandez toujours à l'IA d'écrire ou de modifier les classes dans models.py, puis exécutez localement python manage.py makemigrations.

5 Conseils Clés pour le Co-développement

1. Donnez toujours le contexte complet de la stack technique

Lorsque vous ouvrez une conversation avec une IA, commencez par lui présenter brièvement le projet pour obtenir des réponses précises :

Contexte initial (à copier) 
Nous travaillons sur un projet d'application web de mentorat nommé IFRI_MentorLink.
La stack technique est : Django 6 (Python 3.10+), base de données MySQL 8.0, et le frontend utilise du HTML5 sémantique avec Tailwind CSS importé par CDN dans un fichier base.html parent.
Le projet est modulaire avec 3 apps Django : accounts (authentification), profiles (profils utilisateurs), et matching (mise en relation).
Voici le code de mon modèle actuel dans models.py [...]
2. Demandez des tâches atomiques et découpées

Ne demandez pas : "Génère-moi l'algorithme de matching".
Demandez plutôt : "Voici mes deux modèles OffreMentorat et DemandeMentorat. Écris une fonction Python autonome qui prend ces deux instances en paramètres et calcule un score de correspondance de 0 à 100 basé sur les matières communes et les disponibilités.". Cela évite les bugs et le code inutile.

3. Utilisez l'IA pour générer vos jeux de tests et fixtures

Les IA sont excellentes pour générer des données de test réalistes. Donnez-leur votre modèle Django et demandez-leur de créer un script de peuplement (seeding) ou un fichier de fixture JSON pour alimenter la base de données lors de vos démonstrations devant l'encadrant.

4. Maîtrisez le débuggage de Traceback Django

Si Django affiche une page d'erreur jaune, ne paniquez pas. Copiez l'intégralité du Traceback (les lignes d'erreurs en bas de la page) et donnez-le à l'IA en lui montrant le code de la vue concernée. Elle identifiera l'erreur de syntaxe ou le champ manquant instantanément.

Collaborer sans conflits

Base de Données & Workflow Git

La gestion collaborative d'une base de données sous Django peut s'avérer complexe en équipe. Voici le protocole à respecter pour éviter tout dysfonctionnement.

Le protocole de migration Django

Chaque membre travaille sur la branche Git de son groupe. Les fichiers de migration générés localement doivent être commités pour être partagés.

Étape 1 : Créer une modification de table locale

Après avoir modifié vos classes dans un fichier models.py, exécutez les deux commandes suivantes dans le dossier backend/ :

Bash / PowerShell 
python manage.py makemigrations
python manage.py migrate

N'oubliez pas d'ajouter les fichiers créés sous migrations/ dans votre commit Git !

Étape 2 : Récupérer le code de ses collègues

Lorsque vous effectuez un git pull et récupérez de nouvelles migrations, appliquez-les sur votre base MySQL locale immédiatement avant de lancer le serveur :

Bash / PowerShell 
python manage.py migrate
Étape 3 : Résoudre un conflit de migrations (Merge)

Si deux groupes ont généré des migrations parallèles avec le même index de fichier, Django signalera un conflit de branches. Pour résoudre ce problème :

Bash / PowerShell 
# 1. Fusionner les fichiers de migration en conflit
python manage.py makemigrations --merge

# 2. Appliquer la migration de fusion générée
python manage.py migrate
