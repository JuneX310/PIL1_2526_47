handshake 2. Mode de Fonctionnement Interne

Afin de mener à bien le projet sous une contrainte de temps stricte de 10 jours, notre groupe a adopté une approche inspirée de la méthodologie Agile Scrum :

Démarche collaborative et outils

  • Gestion des Tâches (Trello) : Un tableau Trello a été configuré avec les colonnes traditionnelles (Backlog, En cours, À valider, Terminé) permettant de suivre en temps réel la progression de chaque module.
  • Dépôt Git & GitHub : Collaboration obligatoire via Git. Le dépôt PIL1_2526_47 a été partagé avec tous les membres et les enseignants. Nous avons utilisé des branches thématiques pour chaque fonctionnalité (ex: feature/auth, feature/matching) et des revues de code lors des fusions.
  • Rencontres de coordination : Synchronisations quotidiennes de 15 minutes en présentiel ou sur Discord pour lever les points de blocage et planifier les étapes suivantes.

schema 3. Conception et Architecture

Structure Globale du Projet

Le projet est organisé suivant une architecture client-serveur découplée permettant une modularité maximale :

  • Backend (Django) : Contient la logique d'administration, le traitement de la base de données, la messagerie et l'algorithme de matching. Structuré en 3 applications modulaires :
    • apps.accounts : Authentification, profils étudiants, gestion des accès.
    • apps.matching : Offres et demandes, calcul et tri par compatibilité.
    • apps.profiles : Messagerie instantanée, conversations et messages.
  • Frontend (HTML/CSS/JS) : Logé dans le répertoire frontend/. Le style est moderne et responsive (utilisation de variables CSS, flexbox, grid, et de Tailwind pour les composants). L'interactivité est assurée par Vue.js 3 (montage direct pour éviter le besoin d'un build complexe).

Base de données relationnelle (MySQL)

La base de données finale est structurée sous MySQL (tel que recommandé). Les principales relations sont :

  • Une table accounts_etudiant liée par relation un-à-un à la table utilisateur système de Django (auth_user).
  • Une table matching_annonce liée à un étudiant auteur, enregistrant les compétences cibles, le format et les disponibilités spécifiques.
  • Une table matching_match qui lie un mentor et un mentoré avec un score calculé de compatibilité.
  • Des tables profiles_conversation et profiles_message pour stocker l'historique des conversations.

database Fichier SQL de structure

La structure SQL finale et complète a été exportée et se trouve à la racine du dossier de base de données : database/schema.sql.

terminal 4. Guide de Déploiement

Suivez les étapes ci-dessous pour déployer l'application localement en utilisant l'environnement virtuel inclus dans le backend.

1. Cloner le dépôt et se placer dans le projet

git clone https://github.com/votre-compte/PIL1_2526_47.git
cd PIL1_2526_47

2. Configuration de l'environnement

Un environnement virtuel est déjà configuré dans le dossier backend/venv/. Pour l'activer :

# Sur Windows (PowerShell)
cd backend
.\venv\Scripts\activate

# Sur Linux / macOS
cd backend
source venv/bin/activate

3. Installation des dépendances

pip install -r ../requirements.txt

4. Configuration des variables d'environnement (Base de données)

Créez un fichier .env dans le dossier backend/ inspiré de .env.example :

SECRET_KEY=votre_cle_secrete
DB_NAME=mentorlink_db
DB_USER=votre_utilisateur_mysql
DB_PASSWORD=votre_mot_de_passe
DB_HOST=localhost
DB_PORT=3306

5. Application des migrations et lancement

# Appliquer les schémas à MySQL
python manage.py migrate

# Lancer le serveur de développement
python manage.py runserver

L'application sera accessible à l'adresse suivante : http://127.0.0.1:8000/

menu_book 5. Manuel d'Utilisation

1. Inscription et Création de Profil

Rendez-vous sur la page d'inscription. Vous devez remplir un formulaire en 3 étapes :

  1. Étape 1 : Saisissez votre prénom, nom, numéro de téléphone (WhatsApp), genre, filière d'étude à l'IFRI (GL, IA, SI, IM) et votre niveau (L1 à M2). Note: Seuls les emails finissant par @ifri-uac.bj sont acceptés pour valider l'authenticité de l'étudiant.
  2. Étape 2 : Sélectionnez graphiquement vos points forts (compétences acquises) et vos besoins de soutien (points faibles).
  3. Étape 3 : Renseignez vos plages horaires de disponibilité habituelles dans une grille interactive (Matin, Après-midi, Soir du Lundi au Samedi).

2. Connexion

Connectez-vous via l'écran d'accueil avec vos identifiants. Vous pouvez utiliser indifféremment votre adresse email institutionnelle ou votre numéro de téléphone pour vous connecter.

3. Exploration des offres et demandes

L'onglet "Explorer" liste toutes les annonces de la communauté. Vous pouvez :

  • Basculer entre les onglets "Je propose du mentorat" (Offres) et "Je recherche du mentorat" (Demandes).
  • Rechercher par texte libre (nom d'étudiant, descriptif ou compétence).
  • Filtrer instantanément par matière/compétence, par format (présentiel, en ligne), par filière ou par niveau d'étude.

4. Matching Automatique (Vos Matchs)

L'onglet "Mes Matchs" calcule et trie les profils compatibles avec le vôtre. L'algorithme analyse vos points faibles par rapport aux points forts des autres, vos proximités académiques et vos horaires. Un indicateur circulaire affiche votre score de compatibilité en pourcentage pour chaque étudiant.

5. Messagerie Instantanée

Depuis le profil d'un étudiant ou la liste des matchs, cliquez sur "Contacter" pour lui envoyer un message d'introduction. La messagerie intégrée permet d'échanger en temps réel (actualisation automatique toutes les 10 secondes) et de conserver l'historique des conversations pour planifier vos sessions.
