handshake 2. Mode de Fonctionnement Interne

Notre équipe s'est structurée de manière agile avec des rôles et des responsabilités claires, permettant de mener à bien ce projet dans le respect du délai imparti de 10 jours.

Méthodologie collaborative et outils

  • Gestion de projet Agile (Trello) : Un tableau de suivi Kanban a permis d'affecter les tâches et de suivre leur état (Backlog, En Cours, À Valider, Terminé).
  • Réunions d'avancement (Google Meet) : L'équipe a tenu des réunions d'alignement régulières sur Google Meet pour faire le point sur la progression de chaque sous-groupe, identifier les difficultés et planifier les étapes suivantes.
  • Assistance à la conception (Gemini) : Le modèle d'IA Gemini a été activement utilisé comme assistant de projet pour la conception ergonomique, la génération et le raffinement des maquettes graphiques de l'application, l'aide à la structuration du code, ainsi que la rédaction de la documentation finale (ce présent rapport technique HTML et le fichier README.md).
  • Gestion de version Git (GitHub) : Collaboration centralisée sur le dépôt PIL1_2526_47. Chaque sous-groupe disposait de sa branche dédiée (groupe_1, groupe_2, groupe_3) pour éviter les interférences. Les fusions se faisaient sur la branche d'intégration dev puis vers main.

Le Rôle de Facilitateur du Chef de Groupe (SOZAN Euloge)

En complément de son travail de développement dans le groupe 1, le chef de projet a piloté l'intégration globale :

  1. Amorce du Projet : Conception intégrale de la maquette de l'interface utilisateur de A à Z avec l'aide de Gemini, et établissement de la répartition initiale des modules entre les membres.
  2. Architecture de base : Initialisation de la structure physique du projet et de l'arborescence Git (Django côté backend, templates et assets côté frontend).
  3. Supervision active : Contrôle continu de l'avancement des tâches en temps réel et intervention directe pour aider les membres rencontrant des blocages techniques.
  4. Intégration & Résolution des Conflits : Rapatriement de l'ensemble des branches, réalisation de la fusion finale des codes sources de chaque sous-groupe, et gestion manuelle des conflits de fusion (fichiers de code et migrations de base de données).
  5. Rédaction de la documentation : Réalisation de la mise en forme et de la rédaction de la documentation technique (rapport HTML et README.md).

schema 3. Conception et Architecture

Structure Physique du Projet

Le projet adopte une architecture client-serveur avec une séparation nette entre le serveur de services Django et l'interface utilisateur gérée via des modèles HTML dynamisés par Vue.js :

PIL1_2526_47/ ├── README.md # Documentation globale du dépôt ├── requirements.txt # Dépendances Python requises │ ├── frontend/ # Couche de présentation (HTML, CSS, JS) │ ├── static/ # Assets statiques │ │ ├── css/ # Fichiers styles (connexion.css, explorer.css, profil.css, etc.) │ │ └── js/ # Scripts (explorer.js, modifier_profil.js, etc.) │ └── templates/ # Pages HTML modulaires │ ├── base.html # Gabarit de structure globale │ ├── groupe1/ # Pages : explorer.html, matchs.html, publier.html, profil_de_l_etudiant.html, notifications.html │ ├── groupe2/ # Pages : landing_page.html, connexion.html, inscription.html, r_initialisation.html │ └── groupe3/ # Pages : dashboard.html, messagerie.html, mon_profil.html, modifier_profil.html │ ├── backend/ # Logique métier (Django) │ ├── manage.py # Point d'entrée de commandes Django │ ├── config/ # Répertoire de configuration (settings, urls, wsgi) │ └── apps/ # Applications logicielles internes │ ├── accounts/ # Gestion des comptes utilisateurs et profils (models.py, views.py) │ ├── matching/ # Système d'annonces et algorithme de suggestion │ └── profiles/ # Messagerie instantanée et tableau de bord │ └── database/ └── schema.sql # Structure exportée de la base de données relationnelle finale

Base de données relationnelle (MySQL)

La persistance des données repose sur MySQL. Le diagramme ci-dessous illustre l'organisation des principales tables :

  • auth_user : Table native de Django pour la gestion de la sécurité, des droits d'accès et de l'authentification (username, email, hash du mot de passe).
  • accounts_etudiant : Extension d'informations liée par relation One-to-One à auth_user. Contient le numéro de téléphone, le sexe, la filière (GL, IA, SI, IM, SE), le niveau (L1 à M2), la biographie, ainsi que les listes de compétences et créneaux sous formats JSON.
  • matching_annonce : Contient les publications d'offres ou demandes de mentorat créées par un étudiant (matières ciblées, format : présentiel/en ligne, descriptif).
  • matching_match : Table pivot associant un mentor et un mentoré avec un score d'adéquation calculé et un état (en attente, accepté, refusé).
  • profiles_conversation & profiles_message : Tables supportant le canal de messagerie privée, associant deux étudiants et stockant les messages datés et leur état de lecture.

Le schéma relationnel SQL complet et finalisé a été exporté à la racine du module base de données : database/schema.sql.

terminal 4. Guide de Déploiement Local

Pour faire fonctionner le projet sur un serveur de développement local, veuillez dérouler la séquence de commandes suivantes :

1. Clonage du code source

git clone https://github.com/IFRI-PROJETS/PIL1_2526_47.git
cd PIL1_2526_47

2. Initialisation de l'environnement virtuel Python

Créez et activez un environnement isolé dans le répertoire backend/ :

# Accéder au dossier backend
cd backend

# Création de l'environnement virtuel venv
python -m venv venv

# Activation de l'environnement (Windows)
.\venv\Scripts\activate

# Activation de l'environnement (Linux/macOS)
source venv/bin/activate

3. Installation des paquets requis

pip install -r ../requirements.txt

4. Fichier d'environnement (.env)

Dans le dossier backend/, créez un fichier nommé .env en copiant le fichier exemple .env.example, puis ajustez vos paramètres de connexion MySQL :

SECRET_KEY=votre_cle_de_securite_django
DB_NAME=mentorlink_db
DB_USER=root
DB_PASSWORD=votre_mot_de_passe_mysql
DB_HOST=localhost
DB_PORT=3306

5. Migration et démarrage de l'application

Exécutez la mise en place des tables dans votre base MySQL locale et lancez le serveur :

# Appliquer les migrations de structure
python manage.py migrate

# Démarrer le serveur local
python manage.py runserver

Le site est alors disponible à l'adresse suivante : http://127.0.0.1:8000/

menu_book 5. Manuel d'Utilisation

1. Accès & Gestion du Compte (Groupe 2)

  • Découverte (Landing Page) : La page d'accueil présente le concept IFRI MentorLink et oriente les utilisateurs vers la connexion ou l'inscription.
  • Création de compte (Inscription) : L'enrôlement s'effectue en 3 étapes guidées :
    1. Coordonnées de base : Prénom, nom, adresse email (tous les domaines de messagerie sont acceptés), téléphone WhatsApp, filière et niveau d'étude.
    2. Aptitudes & Lacunes : Sélection interactive des matières maîtrisées (points forts) et des matières où un soutien est recherché (points faibles).
    3. Emploi du temps : Sélection des plages de disponibilités hebdomadaires (Lundi au Samedi divisé en Matin, Après-midi et Soir).
  • Connexion : Authentification sécurisée via l'adresse email ou le numéro de téléphone combiné avec le mot de passe.
  • Récupération : En cas de perte du mot de passe, un module permet la réinitialisation sécurisée des accès.

2. Tableau de Bord & Profils (Groupe 3)

  • Le Dashboard : Centre de contrôle affichant les statistiques clés de l'utilisateur (annonces actives, conversations ouvertes, demandes reçues), des recommandations de mentors basées sur l'algorithme, ainsi que le flux des notifications récentes.
  • Mon profil : Permet d'afficher la fiche de profil telle que vue par les autres membres (bio, compétences, grilles horaires, etc.).
  • Modifier le profil : Permet de mettre à jour la biographie, de téléverser une photo de profil ou de couverture, et d'ajuster dynamiquement ses compétences ou disponibilités.

3. Module d'Appairage (Groupe 1)

  • Annonces (Explorer) : Permet de visualiser toutes les publications de soutien de la communauté. Un sélecteur permet de basculer entre les "Offres de mentorat" et les "Demandes de mentorat". Des filtres dynamiques par matière, format (présentiel / en ligne / les deux) et recherche plein texte sont disponibles.
  • Publier : Formulaire rapide permettant de soumettre une nouvelle annonce en choisissant les compétences, le descriptif et le format de session désiré.
  • Matching de profils : La page "Matchs" exécute un algorithme d'appairage. Le score (en %) dépend de trois facteurs : la complémentarité compétences/lacunes, la proximité des filières d'étude à l'IFRI, et le recoupement des créneaux horaires disponibles.
  • Messagerie instantanée : Un bouton de contact ouvre une fenêtre de messagerie privée. Les échanges se mettent à jour automatiquement toutes les 10 secondes pour offrir une conversation fluide.
  • Notifications : Un flux dynamique informe l'étudiant en temps réel de la réception de nouveaux messages ou de demandes d'appairage en attente d'acceptation.
